Getting Started

How to start the y2 cluster.

On every node that you plan to use as nodes, pull the communication service and the worker service from GitHub (see the Download section).

Start the communication service first using sbt run "y2 node --seed-ip ip (where you replace ip with the IP address of one of the node in the cluster).

Then start the worker service on all nodes using python3 main.py.

Finally, start the client service using sbt run "y2 client --seed-ip ip --node-count n" (replace ip with the IP address of one of the node in the cluster and n with the total number of nodes in the cluster).

That's it, the cluster has started and is training your neural network!

How it Works

The theory behind the magic.

The main problem when parallelizing the training of densely connected neural networks is that communicating the full gradient after each epoch is a huge communication bottleneck (especially when the number of node increases).

To solve this issue, the y2 nodes only send a predefined delta value to apply to a given weight. This delta value is known by all nodes in the cluster. The only information that actually needs to be transmitted is whether the delta is positive or negative and the weight to apply the delta to. This can reasonably be done using 4 bytes of data, thus rastically reducing the amount of communication.

Experiments have proven that such compression does not prevent the neural network from converging.

Our implementation is based on a paper by Amazon.
